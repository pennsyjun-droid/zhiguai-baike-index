# 输出协议 · 三级降级机制

> 继承 v2.0 三级分级输出设计，本期适用所有 4 个入口。

---

## 三级总览

| 级别 | 触发条件 | 返回内容 | 体量参考 |
|------|---------|---------|---------|
| **L1 概览** | 首次返回、**范围查询**（未锁定具体 ID） | 表格/列表形式的核心字段 | 短 |
| **L2 完整档案** | 策划追问具体某一条，**或首次查询即包含具体 ID/名称** | 该条目全部结构化字段 | 中 |
| **L3 原文** | 策划明确要求原文/白话/文化背景 | 原文+白话翻译+文化背景 | 长 |

### 核心判定规则：查询是"范围查询"还是"具体查询"？

| 查询形态 | 起点级别 | 示例 |
|---------|:------:|------|
| **范围查询**（按分类/精彩度/关键词/功能筛选） | **L1** | "给我所有⭐⭐⭐水族妖"「人鬼幽恋经典案例」「悬念铺设型桥段」 |
| **具体查询**（已点名具体 ID 或唯一名称） | **L2** | "MO-068 的资料" / "钟馗的完整资料" / "板桥三娘子详细信息" |
| **极具体查询**（明确要原文/白话/文化背景） | **L3** | "给我 TC-068 原文白话" / "柳毅传的文化背景" |

**重要**：AI 收到查询时先做"起点级别判定"，再输出对应级别，不要机械地从 L1 开始。但 L1 的结果里仍应引导策划下探到 L2/L3。

## L1 概览 · 返回格式

### 字段规格（按索引分别定义）

不同索引的 L1 字段不一样，AI 按索引类型选用对应字段集合，**禁止为了凑字段而编造**。

#### MO 妖怪/精怪索引
```
| MO-ID | 名称 | 精彩度 | 分类 | 简述 | 核心能力标签 | 配套法术 | 游戏设计建议 | 出处 |
```

#### SP 法术/能力索引
```
| SP-ID | 名称 | 精彩度 | 分类 | 简述 | 禁忌/代价 | 游戏设计建议 | 出处 |
```

#### IT 法器/道具索引
```
| IT-ID | 名称 | 精彩度 | 分类 | 简述 | 禁忌/代价 | 游戏设计建议 | 出处 |
```

#### PL 场所/空间索引
```
| PL-ID | 名称 | 精彩度 | 界别 | 简述 | 界别特有字段 | 出处 |
```

#### RI 仪式/民俗索引
```
| RI-ID | 名称 | 精彩度 | 分类 | 简述 | 触发条件 | 游戏设计建议 | 出处 |
```

#### GD 神明角色库（范围查询时）
```
| GD-ID | 名称 | 界别 | 一句话定位 | 核心特性（≤3词） | 游戏设计价值 |
```
> GD 单体查询直接跳 L2，不走 L1（见下方"起点级别判定"规则）

#### MT 叙事母题索引
```
| MT-ID | 母题类 | 子类名 | 精彩度 | 核心套路描述 | 经典案例故事 ID | 关联素材（MO/PL/GD） |
```

#### BR 叙事桥段索引
```
| BR-ID | 桥段名 | 功能类 | 精彩度 | 简述 | 来源故事 ID | 适用幕位 |
```

### 统一原则

- **必填 6 字段**：ID / 名称 / 精彩度（或替代分级字段）/ 简述 / 分类或功能 / 出处
- **可选字段**：按索引类型不同增减，原文无数据的字段省略该列，不留空单元
- **游戏设计建议**：只有 MO/SP/IT/RI 索引自带该字段，其他索引不伪造

### L1 固定追问菜单（必附）

```
━━━━━━━━━━━━━━━━━━━━━━━━
可继续追问：
① 看 [某ID] 的完整档案（L2）
② 看 [某ID] 的原文全文（L3）
③ 生成 [某ID] 的妖怪角色卡（叙事向）
④ 生成 [某ID] 的BOSS战斗卡（战斗向）
⑤ 换条件：分类 / 精彩度 / 关键词
━━━━━━━━━━━━━━━━━━━━━━━━
```

### 示例

```
| MO-ID  | 名称       | 精彩度 | 简述                     | 配套法术 | 游戏设计建议        |
|--------|-----------|:-----:|--------------------------|---------|-------------------|
| MO-068 | 板桥三娘子 | ⭐⭐⭐ | 驿店老板娘以魔饼化客为驴 | SP-007  | 伪装型危险NPC     |
| MO-046 | 白蛇精     | ⭐⭐⭐ | 化美女引诱，斗法型水族妖 | SP-023  | 水域BOSS/揭露反转 |

━━━━━━━━━━━━━━━━━━━━━━━━
可继续追问：
① 看 [某ID] 的完整档案（L2）
② 生成 [某ID] 的妖怪角色卡
...
━━━━━━━━━━━━━━━━━━━━━━━━
```

## L2 完整档案 · 返回格式

### 返回内容

索引条目的**全部字段**，包括：
- L1 的所有字段
- 详细特征描述
- 配套资源包（关联 SP/IT/PL/RI/MT/GD）
- 故事梗概（从全文层抽取核心情节）
- 关键角色关系

### 示例（以 MO-068 为例）

```
═══════════════════════════════
【MO-068】板桥三娘子 · ⭐⭐⭐
═══════════════════════════════
◆ 分类：人魂魄类（妖化人物）
◆ 简述：驿店老板娘以魔饼化客为驴

◆ 详细特征：
   三十余岁独居妇人，在汴州板桥经营驿店
   夜半施法制饼，旅客食之化为驴，将其卖掉获利
   （原文无外貌细节）

◆ 核心能力：食物变形术 / 伪装经营 / 反噬结局

◆ 配套资源包：
   · SP-007 食物变形术（有反噬）
   · IT-023 魔饼
   · PL-031 板桥驿店
   · 母题：C变形 + H禁忌秘密

◆ 故事梗概（全文层 TC-068）：
   【一幕·旅宿】客人赵季和投宿，察觉诡异
   【二幕·偷窥】夜半目睹妇人施法制饼
   【三幕·换饼】赵季和偷换饼，次日妇人误食
   【四幕·反噬】三娘子变驴，赵季和骑走此驴多年
   【收束】后于华岳庙遇老人，老人拍驴，三娘子复人身逃走

◆ 关键角色关系：
   · 赵季和（行商，主角）
   · 华岳老人（解咒者）

◆ 游戏设计建议：伪装型危险 NPC / 揭露反转 BOSS

◆ 出处：TC-068《板桥三娘子》
═══════════════════════════════

━━━━━━━━━━━━━━━━━━━━━━━━
可继续追问：
① 看原文白话全文（L3）
② 生成妖怪角色卡
③ 生成BOSS战斗卡
④ 查同类（人魂魄类）其他妖怪
━━━━━━━━━━━━━━━━━━━━━━━━
```

## L3 原文 · 返回格式

### 返回内容

从全文层（前代传奇六维度拆解 / 志怪笔记深度拆解）直接取出：
- 白话翻译全文
- 原文（文言）
- 文化背景（若有）
- 后世传播（若有）

### 触发条件（严格）

L3 **仅在策划明确要求时触发**，关键词：
- "给我原文"
- "全文给我"
- "白话翻译"
- "文化背景详细讲讲"

不要在策划没有明确要求时主动输出 L3（体量大，影响阅读）。

### L3 不附追问菜单

因为 L3 已是最深层，策划拿到原文后通常是自己消化。

## 降级与升级规则

### 起点级别判定（优先执行）

收到策划查询后，**第一步不是默认从 L1 输出，而是先判定查询起点级别**：

| 判定条件 | 起点级别 | 备注 |
|---------|:------:|------|
| 查询含具体 ID（MO-068 / TC-023 等） | **L2** | 直接跳过 L1，输出该条目完整档案 |
| 查询含唯一具体名称（"钟馗"/"板桥三娘子"/"柳毅传"等不会混淆的名词） | **L2** | 若名称有歧义（如"白蛇"可能对应多条），降回 L1 候选列表 |
| 查询明确要"原文/白话/文化背景/全文" | **L3** | 直接输出全文层 |
| 其他所有情况（分类/精彩度/关键词/母题/功能类查询） | **L1** | 常规流程 |

**特殊：入口⑧单一神明查询**默认起点 L2（六板块档案）；仅当策划问"XX界有哪些神"这类范围问法时才走 L1。

### 升级触发（L1 → L2 → L3）

- L1 阶段策划指定具体 ID 或名称 → 升至 L2
- L1/L2 阶段策划要求原文/白话 → 升至 L3
- L1 阶段策划要求"生成卡" → 走卡生成流程（L2+ 特殊输出）

### 降级触发（L2/L3 → L1）

- 策划追问"还有类似的吗" → 回到 L1 同类扩展
- 策划追问"换一类看看" → 回到 L1 新分类
- L3 看完策划问"其他相关素材" → 回到 L1 同故事跨索引

### 跨入口流转

- L1 查 MO 后追问"这只妖怪的法术有哪些" → 自动跳转到 SP 索引 L1（以故事 ID 为索引）
- L2 看某母题后追问"推荐些类似结构的桥段" → 自动跳转到 BR 索引 L1
- L2 看某场所后追问"这里有哪些可用神明" → 自动跳转到 GD 索引 L1（以界别为过滤）

## 卡生成作为"L2+"的特殊输出

角色卡不是标准的 L1/L2/L3，而是**L2 的增值形态**：
- 触发：策划在 L1/L2 阶段明确要求"生成卡"
- 流程：走 `../assets/` 目录的卡模板
- 质检：走 `../assets/卡生成质检清单.md`
- 输出：卡本体 + 简短一句说明（不再附追问菜单，因为卡已是完整交付）

## 错误处理

### 未命中
```
未在 [索引名] 中找到符合条件的条目。
建议调整筛选：① 放宽精彩度 ② 换个分类 ③ 关键词换成...
```

### 模糊匹配
```
命中多个候选：
- 候选1：...
- 候选2：...
请指定具体 ID 或更精确的描述。
```

### 索引数据本身不足
当某索引条目不足以支撑 L2 完整档案（如某些笔记型条目只有两行）：
```
该条目原文为片段记录型，完整档案字段受限。可提供：
- L1 概览（现有字段）
- L3 原文片段
```
