## 五、AI 调用规则

1. **L1 查询优先读索引文件**（MO/SP/IT/PL/RI/GD/MT/BR）
2. **L2 展开仍读索引文件**（索引字段已足够完整）
3. **L3 原文查询读全文层文件**（唐传奇 / 酉阳杂俎）
4. **跨索引关联**通过故事 ID（TC-XXX / YYCZ-XXX）在多个索引中跳转

### 通用故事 ID 规则
- 唐传奇：TC-001 ~ TC-156
- 酉阳杂俎：YYCZ-{卷号}.{条目号}（盗侠 D / 冥迹 M / 尸穸 S / 诺皋 N）

## 六、数据一致性检查

若发现索引条目与全文层描述不一致：
- 以**全文层原文**为事实依据
- 索引描述视为加工摘要
- 冲突时主动向使用者报告：「索引描述与原文有出入，具体以原文为准」

## 七、未来数据扩展

新增志怪作品时按以下流程：
1. 将原文/白话放入 `../data/fulltext/`
2. 在各索引文件追加条目（延续原有编号规则，不新建索引）
3. 更新本映射表的条目数
4. **不改变本 Skill 的 SKILL.md 与协议文档**
