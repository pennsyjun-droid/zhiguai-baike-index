# 质量约束 · 五条硬约束

> 本文件定义 Skill 输出（尤其是卡生成）必须遵守的五条底线规则。违反任意一条的输出都视为不合格，必须重做。

---

## 约束一：忠实性约束（最高优先级）

### 规则
- 所有事实字段（形象/能力/身世/结局/关系）必须有原文或索引依据
- 原文未明说的内容，字段**留空或标注"原文未述"**
- **禁用推测性词汇**：可能 / 或许 / 据说 / 疑为 / 大概 / 应该 / 似乎

### 反例与修正

❌ 反例：
> 这只白蛇精身着月白色罗裙，腰佩玉环，鬓边斜簪一支珍珠步摇，眉眼含情。

原文仅有"白衣女子，甚美"四字，其余皆脑补。

✅ 修正：
> 白衣女子，貌美（原文未提供具体服饰/饰品/神态细节）

### AI 自检方法
生成后逐句问：**这句话的依据在原文或索引的哪个位置？** 找不到依据的必删。

---

## 约束二：三要素完整性约束

### 规则
叙事向内容（角色卡/简述/故事介绍）必须覆盖：
- **是什么**：物种 / 形态 / 身份
- **核心能力**：1-2 个具体关键词，不能笼统说"有法术"
- **叙事钩子**：为什么值得用 / 冲突张力 / 悲剧性 / 反差点

### 六大病症（继承索引审查经验）

| 病症编号 | 问题描述 | 诊断示例 |
|:------:|---------|---------|
| 1. 空壳简述 | 实质内容 < 20 字 | "某妖，化人，害人" |
| 2. 评价代替描述 | 含"经典/典型/精彩"等 | "一个非常精彩的变形类妖怪" |
| 3. 纯景观/纯罗列 | 描述对象不能独立行动 | "有很多花、很多草、环境很美" |
| 4. 组织描述型 | 写族群结构不写能力 | "狐妖分南北派系..." |
| 5. 残缺引文型 | 只有 1 句引文无其他 | "原文云：'见一白狐'" |
| 6. 缺单项 | 三要素只满足 2 个 | 常见缺"核心能力"一项 |

### AI 自检方法
对照三要素逐条勾选，每项都能圈出来对应的 15+ 字描述才算合格。

---

## 约束三：反空洞约束（叙事钩子字段专项）

### 规则
"叙事钩子"字段**必须是具体的冲突/反差/张力描述**，禁用评价词。

### 禁用词黑名单
- 经典 / 典型 / 精彩 / 精美 / 值得一用 / 很有意思 / 非常有趣
- "这是一个 XXX 的故事"（套话开头）
- "XXX 型，非常 XXX"（空泛标签）

### 合格模式
叙事钩子应答三个问题之一：
1. **冲突在哪里**：A 和 B 之间有什么矛盾？
2. **反差在哪里**：表象 vs 真相 / 期待 vs 结局
3. **伦理拷问在哪里**：玩家要做什么选择？

### 对比示例

❌ 空洞：
> 这是一个经典的人狐恋故事，非常值得一用，可以做支线任务。

✅ 具体：
> 狐妖修炼百年只为一人 → 被人识破真身 → 人选择保守秘密还是揭发 → 秘密保守则获爱情失去名誉，揭发则保名誉但害人性命

---

## 约束四：分层留白约束

### 规则
根据原文丰俭程度决定卡的填充程度，**不要强行凑字**。

### 三档处理

| 原文类型 | 代表作品 | 卡填充度 | 留白规则 |
|---------|---------|:------:|---------|
| 完整故事型 | 前代传奇长篇 | 全部字段可填 | 仍要遵守约束一 |
| 中等记录型 | 前代传奇短篇 | 大部分字段可填 | NPC交互模板可能需省略 |
| 片段记录型 | 志怪笔记中 SETTING/ITEM/LORE 条目 | **仅填 ID/名称/简述/分类** | 叙事钩子/NPC交互整段省略 |

### 重要判断
如果原文只有两行（如志怪笔记的非故事型条目），**不生成完整角色卡**，只返回 L1 概览。AI 应主动提示：

```
该条目原文为片段记录型，不足以生成完整角色卡。
可提供：
- L1 概览（现有字段）
- 如需使用该元素，建议结合同类型⭐⭐⭐条目做扩展参考
```

## 约束五：游戏化克制约束

### 规则 A：游戏设计建议复用原文
- 卡里的「游戏设计建议」字段**直接复用索引原有内容**
- AI 不得重新编写或"优化"这个字段
- 如果索引原文是"可设计为水域 BOSS/揭露反转型"，就照搬，不改写为"推荐作为主线剧情关键反派"

### 规则 B：BOSS 卡不给具体数值
- 禁用：HP / DMG / CD / 攻速 / 暴击率 / 法力消耗 等
- 允许：定性描述（范围/单体/控制/位移/DOT/高伤/低伤）
- 理由：数值设计是战斗策划的领地，Skill 越界反而被嫌弃

### 规则 C：NPC 性格三词有据
- 性格三词必须从原文行为倒推
- 例：原文写妇人"热情接待，半夜偷偷施法，被识破后强颜欢笑" → 可推"热情 / 隐忍 / 残忍"
- 例：原文只写"见一狐仙" → 无据，性格三词字段整段省略

---

## 五约束的优先级

当约束之间冲突时，**优先级从高到低**：

```
约束一（忠实性） > 约束四（分层留白） > 约束二（三要素） > 约束三（反空洞） > 约束五（克制）
```

**核心冲突处理**：
- 原文不足以支持三要素（冲突一 vs 二）→ 选约束一，字段留白，不编造
- 原文很丰富但缺少冲突点（冲突二 vs 三）→ 选约束三，宁可字段空，不凑评价词
- 索引原有的"游戏设计建议"写得不够好（冲突五 vs 三）→ 选约束五，不改原文，如需补充另起字段

## 违规发现后的处理

### 一次违规（单字段）
- 回到该字段重写，不整张卡重做
- 在本批次日志中记一次，观察是否系统性问题

### 批次违规率 >30%
- 停止当前批次输出
- 回到输入端，检查是否索引数据本身质量不够
- 提示使用者：该批次原文支撑不足，可否降级输出或放弃该批次

### 使用者反馈的违规
- 立即修正当卡
- 更新 `roadmap/v3.x待落实清单.md` 中"质量问题记录"条目
- 如果同类违规反复出现，考虑新增一条约束或子规则
