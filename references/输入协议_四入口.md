# 输入协议 · 四入口使用规范

> 本期开放 4 个策划使用入口。每个入口定义了标准提问句式、参数维度、AI 处理流程。

---

## 入口① 按素材类型查

### 适用索引
MO / SP / IT / PL / RI / GD

### 标准提问句式

```
有哪些 [分类关键词] 类 [素材类型]？
给我 [精彩度] 的 [分类] [素材类型]
[素材类型] 里有没有 [特征描述] 的？
```

### 可组合维度
- **分类**：各索引自带分类（如 MO 的 16 类、PL 的 6 界别）
- **精彩度**：⭐ / ⭐⭐ / ⭐⭐⭐
- **能力/功能关键词**：如"变形""驱邪""治愈"
- **代价/禁忌筛选**：如"有使用代价的""有禁忌的"

### AI 处理流程
1. 识别素材类型 → 定位到对应索引文件
2. 按分类/精彩度/关键词筛选
3. 返回 L1 概览卡（完整表格）
4. 附追问菜单

### 示例

**提问**：给我所有⭐⭐⭐的水域类妖怪

**返回**（L1 概览格式）：
```
| MO-ID | 名称 | 精彩度 | 简述 | 核心能力标签 | 配套法术 | 游戏设计建议 |
|-------|------|:-----:|------|------------|---------|------------|
| ...   | ...  | ⭐⭐⭐ | ...  | ...        | ...     | ...        |

可继续追问：
① 看 [某ID] 的完整档案（L2）
② 生成 [某ID] 的妖怪角色卡
③ 生成 [某ID] 的BOSS战斗卡
```

### 适用索引
MT（叙事母题索引，287 条，13 类）

### 13 类母题速查
A 复仇报应 / B 跨界恋情 / C 变形精怪 / D 仙道修行 / E 英雄侠义 / F 技艺智斗 / G 命运天命 / H 禁忌秘密 / I 婚恋伦理 / J 政治讽喻 / K 信物认亲 / L 入冥 / M 梦境 / N 离魂

### 标准提问句式

```
有没有 [套路/主题] 的经典案例？
[母题关键词] 类故事给我几个参考

1. 识别主题 → 对应母题分类
2. 查该分类下 ⭐⭐⭐ 经典案例优先
3. 返回 L1 格式：故事 ID + 一句话梗概 + 关联 MO/PL/GD
4. 附追问菜单（可展开具体某一故事）

### 示例

**提问**：给我"人鬼幽恋"类的经典案例

**返回**：
```
→ MT 索引 B 类「跨界恋情」子类命中：
  · B012《柳毅传》人龙恋 ⭐⭐⭐（关联 GD-089 洞庭君 / PL-038 洞庭水府）
  · B023《李章武传》人鬼恋 ⭐⭐⭐（关联 MO-xxx / RI-xxx 丧仪送魂）
  · B031《霍小玉传》人妓鬼恋 ⭐⭐⭐（关联 ... ）

可继续追问：
① 看 [某故事ID] 的完整档案
② 生成涉及的 [MO] 妖怪角色卡
```

### 10 类桥段功能
A 悬念铺设 / B 真相揭露 / C 恐怖惊悚 / D 战斗对抗 / E 情感冲击 / F 幽默讽刺 / G 视觉奇观 / H 道德哲思 / I 仪式民俗 / J 余韵留白

### 标准提问句式

```
来个 [功能类] 型的桥段
有没有适合做 [幕位] 的桥段？
[功能关键词] 型的场景单元给我几条
```

### 本期限制
- **不按情绪筛选**（情绪标签尚未批量标注）
- 仅按 10 类功能检索
- 仅按"适用幕位"（铺设/发展/高潮/收束）做辅助筛选

### AI 处理流程
1. 识别功能类 → 查 BR 索引对应类别
2. 返回 L1：桥段 ID + 简述 + 来源故事 + 适用幕位
3. 附追问菜单

### 示例

**提问**：来个悬念铺设型的开场

**返回**：
```
→ BR 索引 A 类「悬念铺设」42 条命中高精彩度：
  · A034 崔思兢反间计全流程（信息差/博弈）
  · A047 夜半琴声（环境异常/跟踪）
  · A049 每曲结尾有人消失（规律异常）
  ...

可继续追问：
① 看 [某桥段ID] 的完整描述和原文出处
② 查该桥段所在故事的其他桥段（同故事打包）

## 入口⑧ 查神明角色

### 适用索引
GD（神明角色库，122 位，5 界别）

### 5 界别速查
天界 43 / 幽冥界 30 / 上古界 13 / 人间界 31 / 水域界 5

### 标准提问句式

```
给我 [神明名] 的完整资料
[界别] 有哪些神？
[功能关键词]（如"审判/战斗/爱情"）类的神明有哪些？
```

### AI 处理流程
1. 识别是"单一神明查"还是"群组查"
2. **单一神明（唯一名称/GD-ID）→ 起点即 L2**（跳过 L1），返回六板块完整档案
3. **群组查询（界别/功能类）→ 起点为 L1**，返回 GD 索引概览表
4. 附追问菜单

> 说明：此处 L2 跳级规则与输出协议「起点级别判定」一致，不矛盾。L1 仍是"范围查询"的默认起点，只是单神明查询不属于范围查询。

### 六板块档案结构
① 基本信息 / ② 形象描述 / ③ 核心特性 / ④ 身世来历 / ⑤ 关联角色 / ⑥ 游戏设计价值

### 示例

**提问**：给我钟馗的完整资料

**返回**：
```
═══════════════════════════════
【GD-007】钟馗 · 幽冥界
═══════════════════════════════
① 基本信息：捉鬼大神，幽冥界执法者
② 形象描述：黑脸虬髯/大袍冠帽/持剑或持扇
③ 核心特性：嫉恶如仇/刚烈/捉鬼除妖/赐福镇宅
④ 身世来历：应举不第/撞殿阶死/皇帝赐绿袍厚葬
⑤ 关联角色：冥判系统同僚 / 门神系统组合
⑥ 游戏设计价值：
   · BOSS型：钟馗试炼（正直度检测）
   · NPC型：捉鬼任务委托人
   · 剧情钩子：情欲与正义对立

可继续追问：
① 生成钟馗的妖怪角色卡（以神明作 NPC 原型）
② 看同类（幽冥界审判型）其他神明
```

## 通用规则

### 对所有入口的统一要求

1. **返回结果必须可回溯**：每条都带 ID 和出处
2. **L1 概览必附追问菜单**：引导策划进入 L2 或卡生成
3. **追问卡生成时，切换到卡模板流程**：走 `../assets/` 目录的模板
4. **无命中时明确说明**：不得伪造数据；告知可能的相关分类

### 策划提问不清晰时

AI 主动追问 2-3 个关键维度，而非猜测：
- 素材类型（查什么）
- 精彩度偏好
- 功能/风格偏好（战斗/情感/氛围）

不要超过 3 个追问，避免对话轮次膨胀。
