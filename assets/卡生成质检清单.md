# 卡生成质检清单

> 每张卡输出前，AI 必须逐项勾选。任一项未通过，禁止输出，需回流程修正。

---

## 通用七项（妖怪卡 + BOSS卡共用）

```
□ ① 所有事实字段都能回溯到原文或索引条目，标注了来源 ID
□ ② 全文没有"可能/或许/据说/疑为/大概"等推测性词汇
□ ③ 三要素齐全（是什么 / 核心能力 / 叙事钩子）
□ ④ 叙事钩子或战斗阶段设计包含具体冲突/情节点，不是评价词
□ ⑤ 原文信息不足的字段已留白或标注"原文未述"，未凑字
□ ⑥ 游戏设计建议复用索引原文，未重新编写
□ ⑦ BOSS 卡的所有技能/机制能映射回 SP/IT/PL 某一条索引（仅 BOSS 卡适用）
```

---

## 妖怪角色卡 · 附加三项

```
□ A. 性格三词能从原文行为倒推，无据则此字段整段省略
□ B. NPC 交互分支基于原文情节可能性，未脱离文本
□ C. 叙事钩子至少 2 条，每条 ≥ 15 字的具体冲突描述
```

---

## BOSS 战斗卡 · 附加三项

```
□ D. 没有任何具体数值（HP / DMG / CD / 攻速 / 暴击率等）
□ E. 阶段切换对应原文真实情节转折（如揭露/反噬/召唤），而非通用"狂暴化"模板
□ F. 技能包每一条都能标注 SP-ID 或原文明确来源
```

---

## 不通过处理流程

1. **一项不通过** → 回到该字段重写，再次全表勾选
2. **三项以上不通过** → 整卡废弃，重新从索引检索开始
3. **出现违规 2 次（同一批次内）** → 提示使用者检查上游数据质量

---

## 使用者反馈通道

使用者发现任何一张输出的卡存在下列问题，请直接反馈：

- 字段脑补（原文没写的内容被编出来）
- 叙事钩子空洞（只有"经典/精彩/典型"等评价）
- BOSS 卡给了具体数值
- 阶段设计脱离原文情节
- 技能无法在 SP 索引找到映射

反馈方式：在对话中直接指出"哪张卡、哪个字段、原文依据是什么"，Skill 会按此迭代质检规则。
